Knapsack Problem
an optimization problem used to illustrate both problem and solution. It derives its name from a scenario where one is constrained in the number of items that can be placed inside a fixed-size knapsack. Given a set of items with specific weights and values, the aim is to get as much value into the knapsack as possible given the weight constraint of the knapsack.
Problem Statement
Given weights and values of n items, put these items in a knapsack of capacity W to get the maximum total value in the knapsack
Brute Force
01 knapsnack is the most basic problem to understand 2-d Dynamic programming.AS we have designed a O1 knapsack calculator ,Here is a brief overview of it Brute force approach-Aim for this problem is to find a subset of weight array which maximizes the profit .so for each entity we have option to whether select it or not. So start with either first or last element for example weight array wt[]={2,7,3,4},weight of knapsack -11 and profit array pr[]={5,20,20,10) Let f(x,y) denote that we have x elements to choose from and y is the weight left in knapsack.so problem can be break down recursively as F(4,11)=max(inc,exc) i.e Inc= price[0]+f(3,11-wt[0]) Exc=f(3,11) As it is a recursive method .it will take lot of computational time .TO solve this problem we will optimize it with dp
DP Approach
In every recursive function call we will pass a dp[][] array . Dp[x][y]=means f(x,y) If dp[x][y]!=-1 then we already have f(x,y) thus saves some computation otherwise call f(x,y).
